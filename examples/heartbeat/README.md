# Пример использования Activity Heartbeats в Temporal (Go SDK)

Этот пример демонстрирует работу механизма **Activity Heartbeats** в Temporal.io с использованием Go SDK.

## Что такое Activity Heartbeats?

**Heartbeat (сердцебиение)** — это периодический сигнал, отправляемый исполняемой Activity назад на Temporal Server. Он решает две важные задачи:
1. **Обнаружение зависаний/сбоев воркера (Heartbeat Timeout)**: Если воркер завис, перегружен или упал без закрытия сетевых соединений, Temporal Server обнаружит отсутствие очередного Heartbeat по истечении `HeartbeatTimeout` и перезапустит Activity на другом воркере (согласно `RetryPolicy`).
2. **Сохранение прогресса (Heartbeat Details)**: При отправке Heartbeat (`RecordHeartbeat`) можно передать полезную нагрузку (например, номер текущей итерации, обработанный смещение или процент выполнения). Если Activity упадет и будет перезапущена, следующая попытка сможет прочитать сохраненный прогресс через `GetHeartbeatDetails` и возобновить выполнение с того же места.

---

## Сценарий демонстрации

1. Инициализируется долгоиграющая задача (`HeartbeatActivity`), состоящая из **10 шагов**. Каждый шаг занимает 1 секунду.
2. В параметрах запуска Activity выставлены следующие таймауты:
   - `HeartbeatTimeout: 2 * time.Second` — если Heartbeat не будет записан в течение 2 секунд, сервер сочтет Activity упавшей.
   - `RetryPolicy` разрешает до 3 попыток выполнения.
3. На **1-й попытке (Attempt 1)** после успешного завершения шага 4 Activity намеренно имитирует зависание: засыпает на 4 секунды (`time.Sleep`), не вызывая `RecordHeartbeat`.
4. Сервер Temporal фиксирует превышение `HeartbeatTimeout` и принудительно отменяет контекст текущей попытки.
5. Сервер инициирует **2-ю попытку (Attempt 2)**.
6. Вторая попытка считывает сохраненные детали предыдущего Heartbeat с помощью `activity.GetHeartbeatDetails`. Она видит, что шаг 4 был успешно завершен.
7. Выполнение возобновляется с **шага 5** и успешно доходит до шага 10.

---

## Как запустить пример

1. Убедитесь, что локальный сервер Temporal запущен (из папки `temporal`):
   ```bash
   make infra-up
   ```
   Web UI будет доступен по адресу http://localhost:8233.

2. Откройте терминал и запустите воркер:
   ```bash
   make run-worker-heartbeat
   ```

3. В другом терминале запустите стартер для инициализации Workflow:
   ```bash
   make run-starter-heartbeat
   ```

4. Наблюдайте логи воркера. Вы увидите, как первая попытка зависает на шаге 4, прерывается, а вторая попытка подхватывает выполнение с шага 5.
